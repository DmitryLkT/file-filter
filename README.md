# :page_facing_up: File Filter Utility

Утилита для фильтрации данных из текстовых файлов по типам: целые числа, вещественные числа и строки.
Данные из одного или нескольких разделяются переводом строки и каждый тип записывается в отдельный выходной файл.

## Функциональность
- Утилита поддерживает несколько входных файлов.
- Выходные файлы создаются только при наличии данных соответствующего типа.
- Можно указать префикс и путь для выходных файлов.
- Присутствует два режима статистики: краткая и полная.
- Обработка ошибок: программа не падает на некорректных данных или недоступных файлах.
- По умолчанию файлы результатов перезаписываются.

## Особенности реализации
- Для целых читсел используется BigInteger, для поддержки очень больших чисел без переполнения.
- Для вещественных чисел используется BigDecimal для точного суммирования.
- Строки проверяются на пустые значения и игнорируются, если пустые.
- Каждый тип данных имеет свой класс статистики и writer, чтобы избежать смешивания данных раазных типов.

## Аргументы
- ***-o <путь>*** - путь к папке для выходных файлов (по умолчанию - текущая директория)
- ***-p <префикс>*** - префикс для имен выходных файлов (по умолчанию пустой)
- ***-а*** - добавлять данные результата в существующие файлы (вместо перезаписи)
- ***-s*** - краткая статистика (кол-во элементов записанных в выходных файлах)
- ***-f*** - полная статистика (для чисел дополнительно содержит min и max значения, сумму чисел и их среднее значение.
Для строк помимо кол-ва содержит также размер самой короткой и самой длинной строки)

## Установка
### Клонировать репозиторий
```
git clone https://github.com/DmitryLkT/file-filter.git
```
### Собрать прорект
```
mvn clean package
```
После сборки в папке ***target*** будет файл ***util.jar***
### Запуск
```
java -jar target/util.jar [аргументы] <пути входных файлов>
```
